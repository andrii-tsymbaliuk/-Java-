## **1. Для чого призначений Collections Framework?**

**Collections Framework** у Java — це набір інтерфейсів, класів та алгоритмів для роботи зі структурами даних.  
Він забезпечує:

- уніфікований підхід до зберігання та обробки даних,
- ефективні алгоритми сортування, пошуку та маніпуляції,
- стандартизовані структури: List, Set, Map, Queue,
- зручність, безпеку та продуктивність роботи з даними.

## **2. В чому полягає різниця між масивами та колекціями?**

| Масив (array)                       | Колекція (Collection)                            |
| ----------------------------------- | ------------------------------------------------ |
| Фіксований розмір                   | Розмір може змінюватися                          |
| Працює тільки з індексами           | Має методи для вставки, видалення, пошуку        |
| Може зберігати примітиви та об’єкти | Зберігає лише об’єкти                            |
| Більш низькорівневий                | Високорівнева абстракція                         |
| Немає готових алгоритмів            | Має сотні готових методів (Collections, Streams) |

## **3. Які особливі властивості є у списків порівняно з іншими типами колекцій?**

Списки (List):

- Зберігають елементи **у певному порядку**.
- Дозволяють **дублікати**.
- Дають доступ за **індексом**.
- Підтримують методи вставки в будь-яку позицію.
- Зручні для послідовного перегляду та маніпуляції елементами.

---

## **4. Чим списки відрізняються від масивів?**

- У списків розмір **динамічний**, у масивів — **фіксований**.
- Списки є **класами**, а масиви — **вбудованою структурою мови**.
- Списки мають багатий функціонал (`add`, `remove`, `contains`…), масиви — лише індексацію.
- Списки можуть змінювати внутрішню структуру (розширятись), масив — ні.

## **5. Чим ArrayList відрізняється від LinkedList? Який з них у яких випадках краще використовувати?**

### **ArrayList**

- Використовує **масив всередині**.
- Швидкий доступ за індексом (O(1)).
- Повільне видалення/вставка в середину (O(n)).
- Краще для:
  - частого читання,
  - доступу за індексом,
  - послідовного проходження.

### **LinkedList**

- Побудований на **подвійно зв'язаному списку**.
- Доступ за індексом — повільний (O(n)).
- Вставка/видалення в початок/середину — швидка (O(1)).
- Краще для:
  - частих вставок/видалень,
  - реалізації черг/стеки (Queue, Deque),
  - великих структур даних із частими модифікаціями.

## **6. Що таке Iterator? Які переваги над класичним циклом for?**

**Iterator** — це об’єкт, який дозволяє ітерувати колекцію без доступу до її внутрішньої структури.

### **Переваги перед класичним for:**

- Не залежить від того, як зберігаються дані всередині.
- Дає безпечне видалення елементів під час обходу (`iterator.remove()`).
- Працює однаково з будь-якою колекцією.
- Уникає помилок, пов'язаних зі зміною розміру колекції під час обходу.

## **7. Що таке типізовані та нетипізовані колекції?**

### **Нетипізовані колекції (до Java 5):**

- Можуть містити об’єкти різних типів.
- Не гарантують безпеку типів.
- Вимагають кастингу під час витягу елементів.

### **Типізовані колекції (Generics):**

- Містять тільки вказаний тип: `List<String>`, `Set<Integer>`.
- Безпека на етапі компіляції.
- Не потребують кастингу.

## **8. Для чого потрібен інтерфейс RandomAccess?**

**RandomAccess** — маркерний інтерфейс, який показує, що колекція підтримує **швидкий доступ за індексом** (O(1)).  
Наприклад:

- `ArrayList` реалізує RandomAccess → швидкий доступ.
- `LinkedList` → не реалізує, доступ повільний.

Алгоритми з `Collections` використовують це для вибору оптимальної стратегії перебору.

## **9. Чим Collection відрізняється від Collections?**

| Collection                             | Collections                                       |
| -------------------------------------- | ------------------------------------------------- |
| Інтерфейс                              | Утилітний клас                                    |
| Описує основні операції над колекціями | Містить статичні методи (sort, reverse, shuffle…) |
| Є основою для List, Set, Queue         | Не зберігає даних, лише оперує ними               |
| API структури                          | API алгоритмів                                    |

Приклади:

```java
Collection<String> list = new ArrayList<>();
Collections.sort(list);
```
