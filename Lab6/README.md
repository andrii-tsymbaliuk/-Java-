# **Контрольні питання та відповіді**

## **1. Що таке immutable об’єкт? Для чого об’єкти класу String зробили імутабельними? Чому клас String задекларований як final?**

**Immutable об’єкт** — це об’єкт, стан якого неможливо змінити після створення.
Будь-які операції, що "змінюють" такий об’єкт, насправді створюють новий екземпляр.

**Чому String імутабельний:**

- **Безпека.** Рядки використовуються у паролях, URL, файлових шляхах, SQL-запитах — зміна могла б порушити безпеку.
- **Потокобезпечність.** Імутабельні об’єкти безпечні автоматично.
- **String Pool.** JVM зберігає строки в пулі, і вони мають бути незмінними.
- **Стабільність hashCode.** Рядки часто використовуються як ключі в HashMap.

**Чому String — final:**
Щоб заборонити успадкування і не дозволити змінювати внутрішню реалізацію, що порушило б його імутабельність.

## **2. Що таке регулярні вирази? Наведіть приклади регулярних виразів.**

**Регулярні вирази (RegEx)** — це спеціальні шаблони для пошуку, фільтрації, розбиття або валідації тексту.

**Приклади:**

- `\d+` — одна або більше цифр
- `[A-Za-z]+` — слово з латинських букв
- `^\+380\d{9}$` — український номер телефону
- `[A-Fa-f0-9]{2}` — один байт у шістнадцятковому вигляді
- `\s+` — один або більше пробілів
- `^[A-Za-z0-9_]{3,16}$` — логін 3–16 символів

## **3. У чому полягає різниця між оператором `==` та методом `equals()`?**

| Порівняння           | `==`                                                  | `equals()`                  |
| -------------------- | ----------------------------------------------------- | --------------------------- |
| Для об’єктів         | Порівнює **посилання**, чи це один і той самий об’єкт | Порівнює **вміст** об’єктів |
| Для примітивів       | Порівнює значення                                     | Не застосовується           |
| Можна перевизначити? | Ні                                                    | Так                         |

**Приклад:**

```java
String a = new String("hi");
String b = new String("hi");

a == b      // false
a.equals(b) // true
```

## **4. Для чого потрібні класи StringBuilder та StringBuffer?**

Вони потрібні для роботи зі змінними рядками без створення великої кількості нових об’єктів (на відміну від String).

### **StringBuilder**

- Дуже швидкий
- Не синхронізований
- Підходить для однопотокових програм

### **StringBuffer**

- Синхронізований (потокобезпечний)
- Повільніший
- Використовується у багатопотокових програмах

## **5. Як найпростіше прибрати пробіли на початку та кінці String?**

У Java найпростіше:

```java
str.trim();
```

У Java 11+ рекомендовано:

```java
str.strip();
```

**strip()** працює з повним набором юнікодних пробільних символів, тоді як **trim()** — тільки з ASCII-пробілом.
